# 06. 进阶：静态生成原理 (Architecture) 🌸

> 你可能好奇，既然是 Vue 项目，为什么不需要后台数据库？文章存在哪里？

## 1. 静态文件生成 (SSG 雏形)

不同于 WordPress 需要 MySQL 数据库，这个博客是**基于文件系统**的。

在项目部署（Deploy）之前，我们会运行一个 Node.js 脚本：`scripts/generate-tree.js`。

```javascript
// generate-tree.js 伪代码
const fs = require('fs');

// 1. 扫描 notes 文件夹
const files = scanDir('notes');

// 2. 生成 JSON 索引
fs.writeFileSync('public/files.json', JSON.stringify(files));
```

这个脚本做的事情就是：把你的硬盘文件夹结构，转换成了一个 `files.json` 文件。

## 2. 运行时获取

当博客在浏览器运行时：
1. `App.vue` 请求 `files.json`，拿到地图。
2. 用户点击文章。
3. `App.vue` 根据地图里的路径，去请求对应的 `.md` 文件。
4. 浏览器拿到 Markdown 文本，用 `marked.js` 把它渲染成 HTML。

这就是为什么这个博客**不需要后端**，只需要一个静态文件服务器（如 GitHub Pages）就能跑起来。

## 3. 优缺点分析

*   **优点**：
    *   **极速**：不需要等待数据库查询。
    *   **免费**：GitHub Pages 托管完全免费。
    *   **安全**：没有数据库，黑客没法注入 SQL。
*   **缺点**：
    *   **SEO 弱**：因为是 SPA，百度爬虫可能抓取不到内容（谷歌没问题）。
    *   **无评论**：静态页面无法处理用户提交的评论（除非接入第三方服务）。

但这作为个人知识库和学习 Vue 的练手项目，是再完美不过的架构了。
