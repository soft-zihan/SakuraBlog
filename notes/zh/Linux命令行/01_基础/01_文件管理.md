# 📂 文件日常管理 (File Management)

这里涵盖了从查看、创建到移动、删除的所有基础文件操作。

## 1. 浏览与查看 (View)

### ls (List) - 列出目录内容
- `ls -l`：列表显示详细信息 (权限/所有者/大小/时间)
> 💡 **Pro Tip**: 尝试使用 **`exa`** (或 `eza`) 替代 `ls`，支持 Git 状态显示、图标和树状视图。
- `ls -a`：显示所有文件 (包含以 `.` 开头的隐藏文件)
- `ls -lh`：配合 `-h` 以人类可读格式 (KB/MB/GB) 显示大小
- `ls -t`：按修改时间排序 (配合 `-r` 可反向排序)

### cd (Change Directory) - 切换目录
- `cd /path/to/dir`：切换到指定目录
> 💡 **Pro Tip**: 尝试使用 **`z`** 或 **`zoxide`** 替代 `cd`，实现智能跳转 (如 `z pro` -> `/home/user/projects`)。
- `cd ..`：返回上一级目录
- `cd ~`：返回家目录
- `cd -`：返回上一次所在的目录

### cat / head / tail - 内容查看
- `cat -n`：显示行号查看内容
> 💡 **Pro Tip**: 尝试使用 **`bat`** 替代 `cat`，支持语法高亮、自动分页和 Git 变更标记。
- `head -n 20`：查看前20行
- `tail -n 20`：查看后20行
- `tail -f`：实时追踪文件追加内容 (常用于看日志)

### less / more - 分页查看
- **`more file`**：经典的分页查看器，只能向后翻页。
- **`less file`**：交互式翻页查看 (空格下一页, b 上一页, / 搜索, q 退出)。它是 `more` 的强大升级版，建议首选。

### tree - 树状显示
- `tree -L 2`：仅显示前2级目录内容

## 2. 创建与管理 (Create & Manage)

### mkdir (Make Directory) - 创建目录
- `mkdir -p`：递归创建多级目录 (如 `a/b/c`)
- **💡 效率神技：大括号扩展 `{}`**
  - `mkdir -p project/{src,bin,docs}`：一键创建项目结构。
  - `touch file{1..5}.txt`：快速创建 file1.txt 到 file5.txt。

### touch - 创建空文件
- `touch file.txt`：创建新文件或更新已有文件的时间戳。

### cp (Copy) - 复制
- `cp -r`：递归复制整个目录。
- **更安全的操作**：
  - `cp -i`：覆盖前**询问确认**。
  - `cp -n` (no-clobber)：**不覆盖**已存在的文件。
  - `cp -u` (update)：仅当源文件较新或目标文件不存在时才复制。
- `cp -p`：复制时保留文件属性 (权限/时间)。

### mv (Move) - 移动与重命名
- `mv -i`：覆盖前询问确认。
- `mv -n`：移动时**防止意外覆盖**。
- `mv a.txt b.txt`：重命名文件。

### ln (Link) - 链接
- `ln -s 源文件 软链接`：创建**软链接** (类似 Windows 的快捷方式)。
  - **注意**：源文件建议使用**绝对路径**，否则移动链接后会失效。

### rm (Remove) - 删除 (⚠️ 极度慎用)
- `rm -i`：删除前**询问确认**。
- `rm -rf`：强制递归删除。**警告**：Linux 没有回收站，此操作不可逆。
- `alias rm='trash'`：**Pro 建议**：安装 `trash-cli` 并设置别名，将删除的文件移动到系统的回收站。

## 3. 统计与进阶技巧

### 排序与搜索 (配合 ls)
- `ls -lhS`：按文件**大小**排序 (大文件在前)。
- `ls -lht`：按**修改时间**排序 (最近修改在前)。
- `ls -R`：递归列出子目录下所有文件。

### wc (Word Count) - 文本统计
- `wc -l file.txt`：统计行数。
- `ls | wc -l`：统计当前目录下有多少个文件。

### file - 识别文件类型
- `file my_script.sh`：显示文件类型 (如 "Bourne-Again shell script")

### stat - 查看元数据
- `stat file.txt`：显示文件的完整信息 (Inode, 权限, 3种时间戳)。

### diff - 比较差异
- `diff file1.txt file2.txt`：逐行比较两个文件的不同之处。
> 💡 **Pro Tip**: 尝试使用 **`delta`** 替代 `diff`，它提供极美观的语法高亮和 Git diff 查看体验。
